# Теория Communication-Efficient Federated Learning

В распределенном обучении время одной итерации складывается из времени вычислений ($T_{comp}$) и времени передачи данных ($T_{comm}$):
$$T_{total} = T_{comp} + T_{comm}$$

В федеративных системах $T_{comm} \gg T_{comp}$, поэтому сжатие данных является приоритетной задачей.

## 1. Квантование градиентов (Quantization)

Квантование преобразует вектор градиентов $g \in \mathbb{R}^d$ в вектор со значениями из малого дискретного набора.

### SignSGD (1-bit Quantization)
Оператор квантования передает только знак каждого элемента:
$$Q(g) = sign(g) \in \{-1, +1\}^d$$
Сжатие: в 32 раза (с 32-битных float до 1 бита).

### Ternary Gradients
Значения квантуются в $\{-1, 0, +1\}$.
$$Q(g_i) = \begin{cases} +1 & \text{if } g_i > \gamma \\ 0 & \text{if } |g_i| \le \gamma \\ -1 & \text{if } g_i < -\gamma \end{cases}$$
Где $\gamma$ — порог отсечения.

## 2. Разрежение (Sparsification)

Метод Top-K оставляет только $k$ элементов с наибольшей абсолютной величиной, обнуляя остальные ($k \ll d$):
$$S(g) = \text{Top-K elements of } g$$
Сжатие: до нескольких порядков (например, передача только 1% весов).

## 3. Компенсация ошибок (Error Feedback)

Простое сжатие может привести к накоплению ошибки и расходимости. Алгоритм с Error Feedback сохраняет остаток сжатия $e$:
1. $v_t = \eta \cdot g_t + e_t$
2. $u_t = Q(v_t)$ (сжатие)
3. $e_{t+1} = v_t - u_t$ (сохранение ошибки)
4. Передача $u_t$.

Это гарантирует, что даже малые градиенты, отсеченные квантованием, в конечном итоге повлияют на модель, когда накопятся в векторе ошибки.

## Применение в трейдинге

Эффективная коммуникация критична при:
- **Обучении по мобильной сети**: Торговые терминалы могут иметь нестабильное соединение.
- **Глобальной агрегации**: Синхронизация между континентами требует минимизации трафика.
- **Масштабировании**: При наличии 100,000+ участников (крауд-алготрейдинг) общие затраты на трафик центрального сервера становятся доминирующим фактором.
